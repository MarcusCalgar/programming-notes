Clean Code

Since programmers read a lot more code than writing it, learning to write clean, easy to read code is very helpful. Not only for other, but also when you have to re-read your own code.

Some good advice:

  • Keep consistent indentation. (tab or double space.
  • Use semicolons (;) even if the language doesn't require it.
  • Keep the line length short. A lot of sources recommend 80 characters.
  • The naming of functions and variables should be consistent. Always use camelCase.
  • Write explanatory comments, but don't abuse them.
  • Avoid large functions.
  • Avoid indiscriminate mixing of coding languages. Like mixing CSS and JavaScript within the HTML structure.
  • Don't use too many external imports as it will slow down the application.
  • Code tells you how, comments tell you why.
