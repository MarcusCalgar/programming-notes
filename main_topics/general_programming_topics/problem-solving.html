Problem Solving

To be successful at problem solving, follow these steps:

  1. Understand the problem: the first step to be able to solve any problem is to define what the problem actually is.
    • Write it down in plain text
    • Draw some diagrams if necessary
  2. Plan: break down the large problem into manageable smaller ones. Some useful questions are:
    • Does it have an interface? What will it look like? What functionality will it have?
    • What inputs will the program work with? Does it get it from the user, or from somewhere else?
    • What is the desired output?
    • Given the input, what needs to be done to get the desired output?
      This is where the algorithm goes. An algorithm is a kind of recipe to solve a particular problem.
  3. Write pseudocode: pseudocode is the logic of the program in natural language. It helps to slow down and give time to really think about what the program has to be doing.
  4. Divide and Conquer: start coding with the smallest and simplest subproblem. Do NOT try to solve the big problem in a single go.

How to Think Like a Programmer - Andy Harris

Takeaways:

  • Learning to program is hard, because you have to be able to describe what you want to the dumb-as-a-rock computer.
  • Write out the concepts first, then convert to code later.
    • A lot of time, the real problem is not understanding the problem well, not not knowing what code to write.
  • Comments are code.
    • Code explains the comments to the computer.
