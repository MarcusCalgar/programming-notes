Understanding Errors

Overview

An error is a built in Object in the language. It has crucial information about what and where went wrong during execution.

  • Calling on a variable that has not been declared "throws" a ReferenceError.
  • An error has a type and a message and it also tells us in which file the error occurred, on which line.
  • The stack trace helps us to understand when was the error thrown. What functions were running, etc.

Common Types of Errors

  • SyntaxError: happens when the code isn't written according to the rules of the language.
  • ReferenceError: happens when a variable hasn't been declared or is misspelled.
  • TypeError: happens when:
    • an operand or argument passed to a function is not the same as the function expects or needs.
    • attempting to modify a value that cannot be changed.
    • attempting to use a value in an inappropriate way.
    When encountering this error, consider the data types of the elements around, maybe it is not what you expect.

Resolving Errors

  • Read the error message carefully, and try to understand it.
  • Google the error. Somebody else probably got it already.
  • Use the debugger, for example in the Chrome DevTools.
  • For simpler errors, use console.log() and other console statements.

Errors vs. Warnings

While errors immediately crash the running of a program, a warning doesn't. These signal potential problems, though usually not that severe. Nonetheless it is recommended to address them if possible.
