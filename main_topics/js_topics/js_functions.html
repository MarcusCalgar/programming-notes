Functions

Basics

  • A function is a reusable block of code.
  • A function that belongs to an object is called a method.
  • To invoke a function use a set of parentheses after its name. E.g.: myFunction();
  • Some functions must have parameters provided to work. Parameters are sometimes called arguments, properties or attributes.
  • Multiple parameters are separated by a comma.
  • Some parameters are optional.
  • You can use default values for optional parameters. E.g.: function hello(name = "Blake"){console.log(`Hello ${name}.`);}
  • A new addition to language, the nullish coalescing operator, ?? treats null and undefined the same way.
    E.g.: alert(value ?? "unknown); If value is defined, it will be used, if it's not-defined, that is, it is either null or undefined, the code after the operator will run.
  • A function is a value, if assigned to a variable. It represents an action.
  • Function Declarations are available everywhere in the block where they are created.

Anonymous Functions

  • Anonymous Functions have no names, and usually get another function as a parameter.
  • This way of creating a function is called function expression.
  • Note: function expressions are not hoisted.

Arrow Functions

  • (number) => {console.log(number*2);}
  • (number) => console.log(number*2);: If the function has only one line, the curly brackets are not necessary.
  • number => console.log(number*2);: If the function has only one parameter, the parentheses are not necessary either.

Callback Functions

  • Callback Functions are functions that are given as parameters to other functions with the expectation that if needed, they will be called in the function they are added to.

Function Scope and Conflicts

  • When creating a function, it's variables that are created in the function only live in the function. That is, they are not accessible for other parts of the program.
  • The top level of scope is called the global scope. These are accessible everywhere.

Function Return Values

  • A return value is a value that the function returns after it is done.
  • return; without a value immediately exits the function.
  • A function with an empty return or without one, will return undefined.

Naming a Function

  • Since functions are actions that usually do something, the name should describe what it does.
  • Since the function name should describe what the function does, it should only do that one thing.
