Strings

String Creation

  • let stringCreation1 = "double quotes";
  • let stringCreation2 = 'single quotes';
  • let stringCreation3 = `embedded ${embeddedString}`; This is called a template literal.

Other rules

  • Escape characters with \, e.g. 'I\'ve got no right to take my place...'
  • Concatenate strings using the + operator.
  • Numeric values in strings can be turned into a number: let thisIsNowANumber = Number(thisIsAString);
  • Numbers can also be turned into strings: let thisIsNowAString = thisIsANumber.toString();
  • JavaScript expressions can be added to template literals. E.g.: `This can be done ${5 + 89}.`.
  • Template literals respect line breaks in the source code.

String Methods

A method is a bit of functionality that is built into the language or into specific data types.

Primitive values, like "John Doe", cannot have properties or methods (because they are not objects). But with JavaScript, methods and properties are also available to primitive values, because JavaScript treats primitive values as objects when executing methods and properties.

Note: All string methods create new strings. That is, strings are immutable.

  • length: Returns the length of the string. E.g.: text.length;
  • replace("whatToReplace", "newText"): Replaces a specified value with another value in the string. Note:
    • replace() doesn't change the string it is called on.
    • replace() returns a new string.
    • replace() only replaces the first match. Use the /g flag with a regular expression to replace all occurrences.
    • replace() is case sensitive by default. Use /i (insensitive) to change that.
  • toUpperCase(): Converts a string to uppercase.
  • toLowerCase(): Converts a string to lowercase.
  • concat(str1, str2): Joins two strings together, similar to str1 + str2.
  • trim(): Removes all whitespace from both sides of the string.
    • trimStart(): Only removes whitespace from the beginning of the string.
    • trimEnd(): Only removes whitespace from the end of the string.
  • padStart(numOfChars, strToPadWith): Pads a string with another string at the start.
  • padEnd(numOfChars, strToPadWith): Pads a string with another string at the end.
  • split("separator"): Converts a string into an array, using the separator as border.
    If no separator is given, the whole string will be in an array at [0].

Extracting String Parts

  • slice(start, end): Returns the extracted part as a new string. Note: the character at the end position isn't included.
    If the second parameter is omitted, the method slices until the end of the string. If the values are negative, positions are counted from the end of the string.
  • substring(start, end): Similar to slice, but values less than 0 are treated as 0.
  • substr(start, length): Similar to above, but the second parameter is for length, rather than position.

Extracting String Characters

  • charAt(position): Returns the character at the specified position.
  • charCodeAt(position): Returns the unicode (UTF-16) of the character at the specified position.
  • Property access [position]: Might be a little unpredictable.
    • Makes a string look like an array, while it is NOT.
    • If no character is found, it returns undefined, while charAt returns an empty string.
    • It is read only. E.g.: str[0] = "A"; doesn't throw an error, but doesn't work.
