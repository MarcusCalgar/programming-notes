JavaScript Arithmetic

  • + - Addition
  • - - Subtraction
  • * - Multiplication
  • ** - Exponentiation
  • / - Division
  • % - Modulus (Remainder)
  • ++ - Increment
    • Post increment does the increment but returns the original number. num++;
    • Pre increment also does the increment, but then returns the updated number. ++num;
  • -- - Decrement
  • JavaScript numbers are 64 bit double precision floating point.
  • Integers are accurate up to 15 digits, while their maximum number can be 17.
  • == : equality: test if the values are equal.
  • === : strict equality: test if the values and the data types are equal.
  • The + sign as a unary operator converts non-numbers to numbers.
    • +true == 1
    • +"" == 0
  • The comma , operator divides expressions, but only returns the last one. E.g.: let a = (1+2, 3+4); alert(a); returns 7.
  • Non-addition arithmetic operators turn empty strings into 0. E.g.: ""-2=-2
  • true + false returns 1.
  • null becomes 0 after numeric conversion.
  • Space characters (/n /t) are converted to 0.
  • Operations with assignment always return undefined, so it is not possible, to assign a value and read it in the same expression.

Adding numbers and strings

  • Adding two numbers result in addition.
  • Adding two strings will concatenate them.
  • Adding a number and a string (or vica versa) results in concatenation.
  • JavaScript will try to convert strings to numbers in all numeric operations, except addition.
  • Trying to do maths with non-numeric strings will result in NaN (Not-a-Number).
  • NaN is a number. (type)
  • Infinity or -Infinity are also numbers. Dividing by 0 returns Infinity.
  • While numbers can be created as Objects, this is erroneous because while an Object Number is equal (==) to a literal, they are not strictly equal (===).
    • Objects cannot be compared.

Some arithmetical methods

  • someNumber.toFixed(numberOfDecimals): rounds someNumber to numberOfDecimals decimal places.
