Call Stack

  • The JavaScript engine uses a call stack to manage execution contexts. There is a global execution context when the main script is run, and function execution contexts for the called functions.
  • The call stack is based on the last-in-first-out (LIFO) principle.
  • When a function is called, it gets to the top of the call stack, and gets popped off when it finished running.
  • The JS engine has one call stack.
