JavaScript Fundamentals

Variables

A variable is like a 'box' for data.

  • Use console.log() to log messages to the console.
  • Use alert(message) to make the website a popup with the content of message.
  • Declare a variable with the let keyword.
  • It is possible to declare variables on a single line, but it isn't recommended. E.g.: let user="John", age=25, message="hello";
  • The keywords var and let work very similarly.
  • The content of a variable can be easily changed. E.g.: let name="John"; name="Roger";
    Note: some functional programming languages like Scala and Erlang forbid the value change of declared variables.

Variable naming

  • A variable name must only contain digits, letters and the symbols $ and _, however, the first character cannot be a digit.
  • Longer variable names should be in camelCase.

Constants

  • Variables that do not change are declared by the const keyword. Such variables cannot be reassigned.
  • Constants that can be hard coded usually written in uppercase. E.g.: const COLOR_RED="#F00";
  • Creating a new variable for a new thing isn't a problem. A couple more lines of code won't affect performance too much.

Data Types

JavaScript is dynamically typed which means that data types exist, but variables are not bound to them.

  1. Numbers
  2. BigInt: for integer numbers larger than 9007199254740991 (253-1) where precision matters. Add n to the end of the number to create.
  3. String
  4. Boolean (logical type): true and false.
  5. null: In JS, this is not a null-pointer, or a reference to a non-existing object. It rather means "nothing", "empty", or "value unknown".
  6. undefined: "value not assigned".
  7. Symbols: closely tied to Objects.

Call Stack

  • The JavaScript engine uses a call stack to manage execution contexts. There is a global execution context when the main script is run, and function execution contexts for the called functions.
  • The call stack is based on the last-in-first-out (LIFO) principle.
  • When a function is called, it gets to the top of the call stack, and gets popped off when it finished running.
  • The JS engine has one call stack.
