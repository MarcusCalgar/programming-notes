Comparisons

  • a < b, a > b : Greater than / less than.
  • a <= b, a >= b : Greater than or equals / less than or equals.
  • a == b, a != b : Equals and not-equals.

Comparison operators return a boolean value, which can be assigned to a variable.

Behavior

  • Strings are compared letter-by-letter. If they are the same, but one is longer, that will be considered greater. E.g.: Will < William.
    However, capitalization matters, so lowercase letters are greater than their uppercase parts.
  • When comparing different types, strings are converted to numbers. Furthermore, true == 1 and false == 0.
  • A regular equality check (==) cannot differentiate between false, 0 and "" (empty string).
    A strict equality operator (===) checks equality without type conversion.
  • null and undefined:
    • null and undefined are equal, but not strictly equal, since they are a different type.
    • For other comparisons, they are converted to Number. null becomes 0, and undefined becomes NaN.
    • However, null > 0 // false, null == 0 // false, but null >= 0 //true. Comparisons convert null to a number, treating it as 0. On the other hand, == is defined in a way, that null only equals undefined, and vica versa, and nothing else.
    • undefined cannot be compared to anything, it will always return false, except when compared to null or undefined.

Conditional Statements (if, else, else if, switch)

  • Use if to specify a block of code to be executed, if a specified condition is true.
  • Use else to specify a block of code to be executed, if the same condition is false.
  • Use else if to specify a new condition to test, if the first condition is false.
  • Use switch to specify many alternative blocks of code to be executed.
  • ternary operator : condition ? run this code : run this instead. Ternary operators can be chained together, however, it isn't recommended, as it is less readable than an equivalent else...if statement.

Logical Operators

In JavaScript there are 4 logical operators:

  • ||: OR : If any of the arguments are true, it returns true. However, if an operand is not a boolean, it is converted to a boolean:
    • 1 is converted to true, while 0 is converted to false.
    • In a chain of OR, the chain returns the first 'truthy' value, or the last one, if all were false.
  • &&: AND: If any of the arguments are false, it returns false. Also, the precedence of && is higher than ||.
  • !: NOT : Converts the expression to a boolean, then returns the inverse. Sometimes a double not (!!) is used to convert an expression to a boolean.
